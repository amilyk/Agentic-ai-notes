## 智能体 AI（Agentic AI）

*概念来源*：Andrew NG 提出，定义为 **基于基础模型（Foundation Model）构建的应用**。

---

### 1️⃣ 非智能体工作流

- **0‑shot**：不提供案例，直接向 LLM 提需求，让其完成任务。

### 2️⃣ 智能体工作流

- 能够 **将任务拆分为多个执行步骤**。
- 通过 **反思（Reflection）** 实现 **版本迭代** 与持续优化。

---

## 3️⃣ 自主性等级

| 等级 | 描述 |
|------|------|
| **低自主性** | 人工预设任务步骤，甚至决定具体实现方式，AI 只负责执行（相当于 P5，能干活）。 |
| **高自主性** | 人类仅设定需求，AI 完成 **决策 → 任务拆解 → 产出**，并具备 **检查、反思** 能力（能自主干活并自我改进）。 |

---

## 4️⃣ 智能体优势

| 维度 | 说明 |
|------|------|
| **性能提升** | 通过任务拆解、反思与优化，Agentic 模式的性能显著高于非 Agentic 模式。 |
| **并行操作** | 例如论文写作：LLM 可并行抓取多个网页关键词，各自总结后再汇总，提升效率。 |
| **模块化** | 
• **模型**：LLM 可随时替换
• **外部工具**：搜索引擎、数据库等可替换
• **推理逻辑**：智能体的思考流程可替换 |

---

## 5️⃣ Agentic 案例

1. **发票处理工作流**
   
   - PDF 识别 → 提取开票信息、到期日、金额 → 录入数据库
2. **回复客户邮件**
   
   - 识别订单信息、问题 → 查询数据库 → LLM 生成回复 → 人工审核 → 发送给用户
3. **通用客服 Agent**
   
   - 库存查询
   - 退货处理
4. **图像处理 Agent**
   
   - 判断航班是否有空位（结合 OCR / 图像识别）

---

## 6️⃣ Agentic AI 的适用场景

| 场景 | 难易程度 |
|------|----------|
| **流程固定、纯文本任务** | 容易实现 |
| **需求不确定、边执行边解决** | 有难度（类似 ReAct） |
| **多模态输入** | 需要多模态支持的框架（如 CAMEL） |

---

## 7️⃣ 关键组成要素

1. **任务分解**
2. **LLM**
   - **文本模型**：任务拆解、决策、信息提取、文本生成等
   - **多模态模型**：OCR、语音转文本等
3. **Tools（API）**
   - 搜索、日历查询、RAG 数据库或知识库检索等
4. **迭代优化能力**
   - 评估 → 改进工作流

---

## 8️⃣ 智能体 AI 评估方法

1. **构建智能体后，人工审核输出**
2. **识别低质量输出**
3. **构建评估指标并追踪错误**（如竞争对手出现的次数与频率）
4. **使用 LLM 进行主观评估**（如论文质量评估）

### 评估类型

- **端到端评估（End‑to‑End evals）**：整体输出质量
- **组件级评估**：单步或单组件的输出质量（如信息抽取准确率）

---

## 9️⃣ 智能体设计模式

### 1. Reflection（反思）

- 对输出进行检查、修改错误、优化，形成 **新版本**。
- 可显著提升性能（纠错 + 版本更新）。
- 评估方式：**主观**（人工）或 **客观**（指标）。
- 反思可以由同一模型完成，也可以使用更高级模型。

### 2. ReAct 模式

- **输入** → LLM 产生 **Thought** 与 **Action**。
- **Action** 调用对应 **Tool**（搜索、OCR、日历等）。
- **Observation**（工具返回）送回 LLM，继续迭代 **Thought → Action**，直至出现 **结束标记**。

### 3. Plan & Solve（计划与执行）

- 根据需求 **自主分解任务** → 生成决策步骤 → 按步骤执行。
- 每一步保留 **原始需求、决策步骤、上一步结果**，用于生成下一步。
- 保证 **多步骤执行不偏离共同目标**。

### 4. 多智能体协作

- 多个智能体协同完成复杂项目，适合 **大规模、复杂任务**。
- **缺点**：调试难度提升。

| 框架 | 适用场景 | 关键特性 |
|------|----------|----------|
| **AutoGEN** | 固定流程、顺序执行 | 多智能体群聊完成，如代码开发 |
| **CAMEL** | 两个智能体深度协作 | 多轮对话、共同任务（写文章） |
| **AgentScope** | 大规模、高并发 | 状态管理、复杂规则、角色扮演类 |
| **LangGraph** | 精准流程控制 | 节点‑边图、条件回退、错误修正 |

---

### 📌 小结

- **Agentic AI** 通过 **任务拆解 + 反思 + 工具调用**，在 **性能、并行、模块化** 上显著优于传统 LLM 工作流。
- 选型时需结合 **任务自主性、输入模态、并发需求**，合理挑选 **AutoGEN / CAMEL / AgentScope / LangGraph** 等框架。
- 评估体系应覆盖 **端到端** 与 **组件级** 两层面，结合 **人工审查** 与 **LLM 主观评估**，形成闭环的 **迭代优化**。
